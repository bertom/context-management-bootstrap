## Pre-Completion Quality Checklist

### Acceptance Criteria
- [ ] All acceptance criteria from brief are met
- [ ] Each criterion has been tested/verified
- [ ] Evidence of verification documented
- [ ] Criteria are independently verifiable

### Code Quality
- [ ] Code follows project style guide
- [ ] Code is properly formatted
- [ ] Linting passes (if applicable)
- [ ] No code smells or anti-patterns
- [ ] Code is readable and maintainable
- [ ] Appropriate error handling implemented
- [ ] Code comments added where needed

### Testing
- [ ] All new tests pass
- [ ] All existing tests still pass
- [ ] Test coverage is adequate
- [ ] Tests are meaningful and well-written
- [ ] Edge cases are tested (if applicable)
- [ ] Integration tests pass (if applicable)

### Brief Compliance
- [ ] All requirements implemented
- [ ] No scope expansion
- [ ] All constraints respected
- [ ] All standards/processes followed
- [ ] Only specified files modified

### Documentation
- [ ] Required documentation updated (if applicable)
- [ ] Version and date updated correctly
- [ ] Documentation accurately reflects implementation
- [ ] Documentation follows project style

### Version/Commit
- [ ] Version bumped correctly (if applicable)
- [ ] Commit message follows format
- [ ] Commit message is clear and descriptive
- [ ] Commit includes coherent change set
- [ ] No secrets in commits

### File Management
- [ ] No temporary files left behind
- [ ] No debug code left behind
- [ ] No commented-out code (unless specified)
- [ ] File structure follows conventions
